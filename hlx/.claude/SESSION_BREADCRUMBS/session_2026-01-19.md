## Discoveries & Learning

**Critical insights:**
1. **Why Claude kept defaulting to Rust:** The bootstrap capsule vision was radically different from procedural syntax. The confusion was real, not just execution.
2. **Nothing was wasted:** RustD proves determinism works. HLX can use RustD as inspiration for LSP, patterns, optimizations.
3. **The real architecture:** Procedural language (HLX) wrapping contract-based values (HLX-D) using dual representation (A and R).
4. **Self-hosting is real:** HLX compiler in HLX compiles itself. This validation proved the language works.

**Questions that emerged:**
- How to embed contract syntax in HLX-A without making it unreadable?
- Should handle syntax `⟁handle_name` be in HLX-A, or only HLX-R?
- What should HLX-D be named? (HLX-D, LCS, ContractSpec, DataLayer?)
- How does mode-switching (⚯) fit into modern HLX?
- Can we do hybrid contract/handle semantics without breaking procedural comfort?

**Rabbit holes we avoided:**
- Didn't go down "fix all Rust" path because we stopped and asked "wait, why am I doing this?"
- Didn't start rebuilding HLX from scratch; instead chose "embed and evolve"
