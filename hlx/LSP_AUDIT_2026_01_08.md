## Gemini's Pain Points → LSP Solutions

### Pain #1: "Reg type missing" (Uninitialized Variables)
**Problem:** LLVM backend requires typed `Store` before `Load`, but compiler emits code that uses registers before initializing them.

**LSP Solution:**
- **Static Dataflow Analysis** - Track variable initialization state per control flow path
- **Diagnostic:** "Variable 'x' may be uninitialized on this path"
- **Quick Fix:** "Initialize 'x' to default value"

**Implementation Plan:**
1. Build CFG (Control Flow Graph) for each function
2. Track variable states: `Uninitialized`, `Initialized`, `MaybeInitialized`
3. Emit warnings for uses before initialization
4. Priority: **P0 (Critical)** - Blocks native compilation

### Pain #2: Binary Format Fragility
**Problem:** Adding instruction variants breaks .lcc compatibility → cryptic errors

**LSP Solution:**
- **Version Hash Checking** - Detect .lcc files with mismatched instruction schemas
- **Diagnostic:** "This .lcc file was compiled with an older HLX version (instruction set v1.2), current is v1.3. Run 'cargo clean && cargo build' to recompile."
- **Quick Fix:** "Recompile all .lcc files"

**Implementation Plan:**
1. Add version hash to .lcc header
2. LSP reads .lcc files in workspace, checks version
3. Warn if mismatch detected
4. Priority: **P1 (High)** - Prevents confusing errors

### Pain #3: Type Mismatches (Int vs Float)
**Problem:** Passing wrong types to functions/contracts caught at compile time, not in editor

**LSP Solution:**
- **Type Inference Engine** - Track types through expressions
- **Diagnostic:** "Expected Float, got Int in call to 'sqrt(x)'"
- **Quick Fix:** "Convert to Float: 'sqrt(x as float)'"

**Implementation Plan:**
1. Build simple type inference system (literals → ops → calls)
2. Check function signatures against inferred types
3. Emit warnings for mismatches
4. Priority: **P0 (Critical)** - Core dev experience

### Pain #4: Missing Semicolons
**Problem:** Parser now shows exact line, but dev has to manually add semicolon

**LSP Solution:**
- **Quick Fix** - "Add missing semicolon"
- **Code Action:** Automatically insert `;` at the right position

**Implementation Plan:**
1. Detect "';' after let statement" errors
2. Offer Code Action to insert semicolon
3. Priority: **P2 (Medium)** - Nice to have, parser already helps
