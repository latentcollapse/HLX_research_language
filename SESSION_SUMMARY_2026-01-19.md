## 🔍 KEY TECHNICAL DECISIONS MADE

1. **Contract Syntax**: `{contract_id:{@field_id:value, ...}}`
   - Rationale: Explicit contract ID and field positioning
   - Alternative: Named fields (deferred for future enhancement)

2. **Field Access**: `.@field_id` operator
   - Rationale: Clear, unambiguous syntax
   - Alternative: `contract.field_id` (deferred)

3. **Collapse/Resolve Keywords**: Added to lexer
   - Rationale: First-class language support
   - Alternative: Library functions (decided against)

4. **VM State as Contract**: Using contract syntax for VM struct
   - Rationale: Demonstrates language features, clean modeling
   - Alternative: Manual array indexing (less elegant)

5. **Fixed Operand Sizes**: All instructions use 4 u32 operands
   - Rationale: Simplifies decoding, wastes bytes but acceptable
   - Alternative: Variable-length encoding (more complex decoder)
