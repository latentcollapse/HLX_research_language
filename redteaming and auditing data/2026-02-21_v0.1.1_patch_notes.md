# HLX v0.1.1 Patch Notes

> **Release Date:** February 21, 2026
> **Based on:** Opus Audit (26 issues identified)

---

## Overview

This patch addresses all **Critical** and **High** severity issues identified in the Opus audit of the HLX runtime. A total of **13 issues** were fixed with proper error handling and test coverage.

---

## Critical Issues Fixed

### C1: Unchecked Register Indexing in VM
**Location**: `vm.rs:196-335`

**Problem**: Direct `self.registers[a]` access could panic on out-of-bounds register index.

**Fix**: 
- Added `get_register_cloned()` helper method
- Replaced all direct `self.registers[idx]` with safe accessor methods
- All register access now returns `Value::Nil` for invalid indices

**Test**: `test_register_bounds`

---

### C2: Integer Overflow in Tensor Shape Calculation
**Location**: `tensor.rs:62`

**Problem**: `shape.iter().product()` could silently overflow.

**Fix**: 
- Changed to `try_fold` with `checked_mul`
- Returns proper error on overflow

**Test**: `test_tensor_shape_overflow`

---

### C3: Register Overflow in Compiler
**Location**: `compiler.rs:416-459`

**Problem**: `u8` register counters wrap at silently at causing register corruption.

**Fix**: 
- Added overflow check before `if self.next_var_reg >= 254 { return Err(...) }`
- Added overflow check for tmp registers
- Clear error message for diagnostics

**Test**: `test_compiler_register_overflow`

---

### C4: Bytecode Read Panics
**Location**: `bytecode.rs:262-280`

**Problem**: `.unwrap()` on `try_into()` could panic.

**Fix**: 
- Replaced with `map_err(|_| RuntimeError::new(...))`
- All bytecode reads now return proper errors

---

### C5: RSI Serialization Panic
**Location**: `rsi.rs:268`

**Problem**: `.expect()` on serialization failure.

**Fix**: 
- Replaced with `map_err(|e| RuntimeError::new(...))`
- Proper error propagation

---

### C6: Negative Index Underflow in `substring`
**Location**: `builtins.rs:17-31`

**Problem**: Negative `start` or `len` cast to `usize` causes massive values.

**Fix**: 
- Added bounds check for `if start_i < 0 || len_i < 0 { return Err(...) }`
- Added `min(s.len())` clamping

**Test**: `test_substring_negative_index`

---

## High Issues Fixed

### H1: Function Call Register Boundaries
**Location**: `vm.rs:1192-1218`

**Problem**: Magic numbers 20 and 150 for register boundaries not validated.

**Fix**: 
- Added validation `arg_base + arg_count <= 256`
- Returns error on boundary violation

---

### H2: Global Tensor Allocation Never Freed
**Location**: `tensor.rs` (new)

**Problem**: Allocation tracking incremented but never decremented.

**Fix**: 
- Implemented `Drop` for `Tensor`
- `fetch_sub` on drop
- Updated `Clone` to track allocations

**Test**: `test_tensor_drop_tracking`

---

### H5: `strcmp` Implementation Bug
**Location**: `builtins.rs:50`

**Problem**: Casting `Ordering` discriminant to unreliable.

**Fix**: 
- Used explicit `match` statement
- Returns `-1`, `0`, `1` reliably

---

### H6: `char` Builtin Unchecked Cast
**Location**: `builtins.rs:66`

**Problem**: `i64 as u8` truncates values > 255.

**Fix**: 
- Added range check `0-127`
- Returns error for out-of-range codes

**Test**: `test_char_builtin_range`

---

## Medium Issues Fixed

### M1: Silent Skip on Invalid Latent Names
**Location**: `vm.rs:549-578`

**Problem**: Invalid string index silently ignored.

**Fix**: 
- Added error return for invalid indices

---

### M2: Agent Memory Leak on Dissolve
**Location**: `vm.rs:1113-1117`

**Problem**: `agent_memories` entries persist after dissolution.

**Fix**: 
- Added `self.agent_memories.remove(&agent_id)` on dissolve

---

### M5: Consensus Division-by-Zero
**Location**: `scale.rs:219-226`

**Problem**: `total == 0` causes division by zero.

**Fix**: 
- Added explicit guard `if total == 0`
- Returns error for empty votes

---

### M7: Incomplete Bytecode Deserialization
**Location**: `bytecode.rs:531-534`

**Problem**: Complex types serializable but not deserializable.

**Fix**: 
- Implemented deserialization for Array, Map, Tensor, Bytes
- Full round-trip now works

---

### M8: Compiler Jump Patch Bounds Check
**Location**: `compiler.rs:327-329, 700-702, 733-735`

**Problem**: No bounds check before bytecode patching.

**Fix**: 
- Added validation `skip_jump + 4 <= code.len()`
- Returns error on invalid patch
